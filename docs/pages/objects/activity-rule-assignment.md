# Activity Rule Assignment (Activity_Rule_Assignment__c)

## Overview

The Activity Rule Assignment object links activity rules to specific users or roles, defining who is responsible for completing activities generated by a given rule. Each assignment can specify the assignment type and include optional notes.

## Object Details

| Aspect | Description |
|--------|-------------|
| API Name | Activity_Rule_Assignment__c |
| Label | Activity Rule Assignment |
| Plural Label | Activity Rule Assignments |
| Sharing Model | Controlled by Parent |

## Fields

| Field Name | API Name | Type | Description |
|------------|----------|------|-------------|
| Name | Name | Text | Primary identifier for the assignment record |
| Activity Rule | Activity_Rule__c | Lookup | Link to the parent Activity Rule |
| Assigned To | Assigned_To__c | Text | User or role the activity is assigned to |
| Assignment Type | Assignment_Type__c | Text | Method of assignment (e.g., User, Role, Queue) |
| Note | Note__c | Text Area | Additional notes about the assignment |

## Relationships

### Parent Objects
- Activity Rule (via Activity_Rule__c lookup)

## Common Use Cases

1. **Activity Distribution**
   - Assigning activity rules to specific users
   - Routing work by role or queue
   - Documenting assignment rationale

2. **Workload Management**
   - Distributing activities across team members
   - Managing assignment types for different rule categories

## Best Practices

1. **Assignment Configuration**
   - Use consistent assignment types
   - Include notes for complex routing logic
   - Review assignments when team composition changes

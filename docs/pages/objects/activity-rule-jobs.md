# Activity Rule Job (Activity_Rule_Jobs__c)

## Overview

The Activity Rule Job object tracks the scheduled execution of activity rules. Each job record represents a planned or completed activity generated by a rule, capturing the assignment, timing, completion status, and notification details.

## Object Details

| Aspect | Description |
|--------|-------------|
| API Name | Activity_Rule_Jobs__c |
| Label | Activity Rule Job |
| Plural Label | Activity Rule Jobs |
| Sharing Model | Controlled by Parent |

## Fields

| Field Name | API Name | Type | Description |
|------------|----------|------|-------------|
| Name | Name | Text | Primary identifier for the job record |
| Activity Date | Activity_Date__c | Date | Scheduled date for the activity |
| Activity Rule | Activity_Rule__c | Lookup | Link to the parent Activity Rule |
| Assigned To Contact | Assigned_To_Contact__c | Lookup (Contact) | Contact assigned to the activity |
| Assigned to User | Assigned_to_User__c | Lookup (User) | User assigned to the activity |
| Cancel with Parent | Cancel_with_Parent__c | Checkbox | If true, job is cancelled when parent is cancelled |
| Completed | Completed__c | Date/Time | Timestamp when the job was completed |
| Move with Parent | Move_with_Parent__c | Checkbox | If true, job date moves when parent date changes |
| Notification ID | Notification_ID__c | Text | Identifier for the associated notification |
| Parent Event ID | Parent_Event_ID__c | Text | Identifier linking to the parent event or task |
| Recommendation | Recommendation__c | Lookup (Recommendation) | Link to a Next Best Action recommendation |
| Type | Type__c | Picklist | Job notification type: Email, Notification, NBA |

## Relationships

### Parent Objects
- Activity Rule (via Activity_Rule__c lookup)

### Related Objects
- Contact (via Assigned_To_Contact__c lookup)
- User (via Assigned_to_User__c lookup)
- Recommendation (via Recommendation__c lookup)

## Common Use Cases

1. **Activity Scheduling**
   - Tracking planned activities generated by rules
   - Monitoring job completion status
   - Managing activity timelines

2. **Notification Management**
   - Configuring email, in-app, or Next Best Action notifications
   - Linking jobs to Salesforce recommendations

3. **Cascading Behavior**
   - Automatically cancelling child jobs when parent is cancelled
   - Moving job dates when parent activity dates shift

## Best Practices

1. **Job Monitoring**
   - Regularly review incomplete jobs
   - Track completion timestamps for SLA reporting
   - Monitor cascading cancellations

2. **Assignment**
   - Assign to either a Contact or User, not both
   - Use notification types appropriate to urgency
